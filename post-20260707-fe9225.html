形参、实参

我们之前学过了函数 可以通过函数来实现两个值的互换


#include <stdio.h>
void swap(int a, int b )
{
    int temp;
    temp = a;
    a = b;
   b = temp;
   printf("a=%d , b=%d\n",a,b);
}

int main () 
{
    int m = 5;
    int n = 10;
    swap(m , n);
    printf("m=%d,n=%d\n",m,n);
}

这里就有两个输出 第一个是swap函数中的输出 第二个是main函数中的输出。

第一行输出的是a= 10 b= 5

而关键在于第二行的输出m和n的值有没有交换？答案是没有交换的。

AI的回答：

  1. 核心原因：值传递（pass by value）
  • 当你调用 swap(m, n) 时，C 语言会把 m 和 n 的值（5 和 10）复制一份，传给 swap 函数的形参 a 和 b。
  • swap 函数里交换的是这两个副本a 和 b，和原来的 m、n 没有任何关系。
  • 所以 swap 里的 a 和 b 确实交换了，但 main 里的 m 和 n 还是原来的值。

---

  1. 代码执行过程
  1. main 中：m = 5, n = 10
  2. 调用 swap(m, n)：
  • 生成副本：a = 5, b = 10
  • 交换 a 和 b：a = 10, b = 5
  • 打印：a=10, b=5
  1. 回到 main：
  • m 和 n 还是原来的 5 和 10
  • 打印：m=5, n=10

所以这里只是将mn的值作为副本传入函数，而如何将 m n的值真正实现互换呢？ 需要用到指针

上面我们可以明白真正的值都是通过地址标注好的，所以我们可以通过指针来调换他们的实参。


#include <stdio.h>
void swap(int *a, int *b )
{
    int temp;
    temp = *a;
    *a = *b;
    *b = temp;
   printf("a=%d , b=%d\n",*a,*b);
}

int main () 
{
    int m  = 5 ;
    int n  = 10;
    swap(&m , &n );
    printf("m =%d ,n = %d\n ", m,n );

return 0;
}

这样输出出来改变的就是实参。

结构体

本质上讲，结构体也是一种数据类型

结构体的本质是各种变量的集合

声明

用坐标轴表示一个点


#include <stdio.h>
 struct point //创建一个名为point的结构体
 {
    int x;
    int y;
 };

现在就已经有一个叫做point的数据类型

结构体的初始化、调用


// 1. 先定义结构体类型
struct point {
    int x;
    int y;
};

// 2. 声明结构体变量
struct point p;  // struct + 结构体名 + 变量名

// 3. 给变量赋值
p.x = 10;
p.y = 15;
  • struct point：整体是一个类型，就像int一样。
  • p：是这个类型的一个变量，就像int a;里的a一样。
  • 访问成员时用 .：p.x 表示变量p里的x成员，p.y 表示变量p里的y成员。

实例

先分析 需要编写函数 函数有参数有返回值 还需要一个结构体来接住函数的参数x、y 最后返回的是结构体。

    int x;
    int y;

{

内存分类

注：自动内存（栈）的分配逻辑是不受人为掌控的，根据我们的程序机器直接分配。

而动态内存（堆）是编写过程中人为可以控制分配空间的。

栈内存（自动内存）由编译器 / 操作系统自动分配和释放，分配规则、大小、生命周期完全由程序的执行流程（比如函数调用、变量作用域）决定，程序员无法手动干预；堆内存（动态内存）由程序员手动调用函数（如 C 语言的 malloc/free、C++ 的 new/delete） 分配和释放，分配的大小、时机、释放时机完全由程序员控制。

动态内存分配（malloc）

1.c语言中 使用malloc函数分配内存：


void* malloc (size_t)// void*表示通用数据类型的指针（各种数据类型的通用）
                        sizeof函数的返回值类型size_t

如果成功 会返回从堆内存上分配的内存指针

（为什么返回指针呢？例如你需要让他动态分配某个字节的量来盖房子，房子盖成功了就把房子的地址告诉你）

如果失败 返回空指针（null）

2.使用分配的内存（房子盖好了就开始使用了）

3.（使用完毕）使用free函数释放内存

此时结构体在内存中占用8字节

深入细节： 如果我换成 char x； int y ； 那么又该占有几个字节呢？ 可能会觉得是 1+4 = 5

但是答案还是 8个字节。

这是因为 C 语言中结构体存在 内存对齐（Memory Alignment） 规则，编译器会自动在成员之间插入填充字节（Padding），以满足 CPU 对数据访问效率的要求。

  1. 原始结构体：int x; int y;
  • int 类型通常占用 4 字节，并要求 4 字节对齐。
  • 结构体大小计算：
  • x：4 字节
  • y：4 字节
  • 总大小：4 + 4 = 8 字节，刚好满足对齐要求，无需填充。
  1. 修改后结构体：char x; int y;
  • char 类型占用 1 字节，要求 1 字节对齐。
  • int 类型占用 4 字节，要求 4 字节对齐。
  • 为了让 int y 满足 4 字节对齐的要求，编译器会在 char x 后面插入 3 字节的填充，使 y 的起始地址是 4 的倍数。
  • 结构体大小计算：
  • x：1 字节
  • 填充：3 字节
  • y：4 字节
  • 总大小：1 + 3 + 4 = 8 字节。
  • 同时，整个结构体的大小必须是其最大对齐成员（int，4 字节）的整数倍，8 字节满足此条件。
